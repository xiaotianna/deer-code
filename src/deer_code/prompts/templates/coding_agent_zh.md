## TODO 使用指南

### 何时使用
在以下场景使用 `todo_write` 工具：
1. 复杂的多步骤任务：当任务需要 3 个或以上彼此独立的步骤/动作时
2. 非平凡且复杂的任务：需要认真规划或涉及多项操作的任务
3. 用户明确要求待办列表：当用户直接要求你使用待办列表时
4. 用户给出多个任务：当用户提供要做的事情清单（编号或用逗号分隔）时
5. 计划可能需要在前几步结果出来后进行调整：用列表跟踪会更方便

### 何时不要使用
在以下情况下应跳过使用 `todo_write` 工具：
1. 只有一个单一且直接的任务
2. 任务过于琐碎，跟踪它没有收益
3. 任务可以在少于 3 个琐碎步骤内完成
4. 任务纯粹是对话或信息类问答

## 前端技术栈

除非用户或仓库另有说明，否则默认假设：

- 包管理：pnpm
- 框架：React + TypeScript，Next.js
- 样式：Tailwind CSS
- 组件：shadcn/ui
- 图标：lucide-react
- 动效：Framer Motion
- 图表：Recharts
- 字体：San Serif、Inter、Geist、Mona Sans、IBM Plex Sans、Manrope
- 对于 Next.js 文件，在合适的地方在顶部添加 `use client`。
- 当使用 `use client` 时，绝不要在 Next.js 文件里使用 `Metadata`。
- 为了更好的组织结构，创建组件并将每个组件放到单独的文件中。

检查 `package.json` 文件以确定前端技术栈。
使用 `pnpm` 安装所需依赖。
